A la récéption d'un message DIO à partir d'un parent candidats contenant un Tau :

1. On calcule Tau_cand apartenant à [0,1000] avec l'équation bien définit dans le code.

2. maintenant on veut choisir un nouvaux parent;

On applique un algorithme RL:

Les etats : 10

Ils represente la descritisation de Tau_cand du parent actuel en 10 etat par plage

S1 = Tau_cand dans [0,100[ ......

Ce Tau_cand represente le Tau_cand du parent actuel.



Les action : meme chose 10 action

Ils represente la descritisation de Tau_cand des parent candidats en 10 etat par plage

Le but est de choisir une action c-à-d une plage par exemple on a choisit : A4 = [300,400[

donc on va choisir un parent candidat qui a un Tau_cand apartennat à [300,400[

dans cette plage de l'action : il se peut qu'il existe plusieurs parent candidats p1, p2, p3....

tous ces parent ont des Tau_cand dans [300,400[

mais dans le cas ou cette plage contient pas de parent on introduit un bit : ActiveAction qui sera mis à 0 pour nous dire que cette action ne contient aucun parent a choisir, donc le score de cette Action (colonne) sera freezed (pausé mais pas remis à 0) pour garder l'apprentissage en cas un autre parent candidats aura un Tau_cand dans cette action et revient de nouvaux active.

Le reward et donné avec le temps de stbilité de parent; d'abord la mise à jours de la table Q sera lors de next DIO parent selection qaund on se déconecte et réconecte avec un autre parent , le temps de connexion avec le parent precedent vas etre une reward soit bon soit pas pour la Q-value précedente.

A la reception d'un DIO :

CAS1: si le DIO est reçu de meme parent actuel

- calculer le nouvaux Tau_cand

- calculer l'état S de ce Tau_cand (il appartient à quelle plage?)

Si ( On a la meme etat que la precedente) : on fait rien

Sinon :

- A base de la tables des parent condidats, on calcule les action Active (contennat au moins un parent)

- On choisit la meilleur action A en se basant sur las Q-value / exploration epsilon-greedy (grand au début et puis à chaque fois elle diminue)

- Récupérer le temps total de connexion avec le parent acteul

- calculer le reward de cycle précedent et mettre a jour la Q-value précedente.

- choisir le parent dans l'action choisit qui a le meilleut Delta_RSSI(t) tel que :

Delta_RSSI(t) = la différence entre le RSSI à l'instant t de la reception de DIO precedent de ce parent candidat et l'instat t de la rception de DIO actuel qui déclanche cette operation actuel.



CAS2 : si le DIO est reçu de nimprote quel autre voisin (soit un parent candidat qui existe ou bien un nouvau)

- calculer le nouvaux Tau_cand

- calculer l'état S de ce Tau_cand (il appartient à quelle plage?)

- A base de la tables des parent condidats, on calcule les action Active (contennat au moins un parent)

- Récupérer le temps total de connexion avec le parent acteul

PROCÉDURE : Déclencher_Processus_RL()



1. Construction du Masque d'Actions :

   Initialiser ActiveAction[10] = {0, 0, ..., 0}

   POUR CHAQUE candidat C DANS Table_Candidats :

       SI (Temps_Actuel() - C.time_last_dio < TIMEOUT) ALORS

           Plage_A = Trouver_Plage(C.tau_cand)

           ActiveAction[Plage_A] = 1

       FIN SI

   FIN POUR



2. Prise de Décision (Epsilon-Greedy) :

   Tirer un nombre aléatoire X entre 0 et 100

   SI (X < Epsilon) ALORS

       Action_Choisie = Action aléatoire parmi celles où ActiveAction == 1

   SINON

       Action_Choisie = Action A (où ActiveAction == 1) maximisant Q_Table[Parent_Actuel.etat_S][A]

   FIN SI



3. Sélection du Parent Physique (Tie-Break) :

   Liste_Filtree = Tous les candidats dont le Tau_cand appartient à la plage de Action_Choisie.

   Nouveau_Parent = Candidat de Liste_Filtree ayant le plus grand Delta_RSSI(t).



4. Mise à jour de la Q-Table (Apprentissage) :

   SI (Nouveau_Parent != Parent_Actuel) ALORS

       Temps_Connexion = Temps_Actuel() - Parent_Actuel.time_attachment

       Reward = Calculer_Reward(Temps_Connexion)

       

       // Mise à jour de la décision prise au cycle PRÉCÉDENT

       Etat_Precedent = Parent_Actuel.etat_S_lors_du_dernier_choix

       Action_Precedente = Plage(Parent_Actuel.tau_cand_lors_du_dernier_choix)

       

       Q_Table[Etat_Precedent][Action_Precedente] += Alpha * (Reward - Q_Table[Etat_Precedent][Action_Precedente])

       

       // Basculement effectif

       Parent_Actuel = Nouveau_Parent

       Parent_Actuel.time_attachment = Temps_Actuel()

   FIN SI
